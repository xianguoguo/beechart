    1.3 一些值得注意的参数

        1.3.1 wmode

            wmode: transparent 可以使图表透明
            wmode: opaque 可以使图表按普通HTML层级定位，否则Flash将浮动在页面最顶层。

        1.3.2 flashvars

            参数都经过flashvar从HTML传递到flash
            有用的参数：
            w / chartWidth
            h / chartHeight
            data
            dataUrl
            css
            cssUrl

2. 如何应用数据

    2.1 加载或解析数据的途径

        2.1.1 parse

            当图表Flash初始化完毕后，外部可以随时调用图表Flash的parse方法
            chart.swf.parse(XML/JSON数据)

        2.1.2 load

            当图表Flash初始化完毕后，外部可以调用图表Flash的load方法
            chart.swf.load(数据路径url)

        2.1.3 flashvars:data

            可以通过flashvars将数据直接输入图表展示。

        2.1.4 flashvars:dataUrl

            如果flashvars中设置了dataUrl，图表初始化时会自动加载该url位置数据文件。

3. 如何自定义样式和皮肤

    与其他Flash图表不同的是，YID Chart使用CSS描述图表的外观。例如在折线图中，
    以下CSS表示线以平滑曲线的方式显示：
    line {
method: curve;
    }

    3.1 加载或解析样式的途径

        有多种方式可以将CSS设置应用于图表Flash中。

        3.1.1 parseCSS

            当图表初始化完毕后，外部可以随时调用图表Flash的parseCSS方法，将样式设置赋予图表渲染。
            也可以在运行时调用该方法，可以动态设置不同的样式。

        3.1.2 loadCSS

            当图表初始化完毕后，外部可以随时调用图表Flash的loadCSS方法，令图表去加载对应url的css文件。

        3.1.3 flashvars:css

            当flashvars中存在css设置时，图表在初始化时会自动渲染这些样式。

        3.1.4 flashvars:cssUrl

            当flashvars中存在cssUrl设置时，图表在初始化时会自动加载对应url的css文件。
